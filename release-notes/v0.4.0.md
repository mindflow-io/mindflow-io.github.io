# Mindflow v0.4.0 — meet the new name, meet the knowledge base

**Released:** 2026-04-30

A two-version-in-one release: this is what would have shipped as v0.3.0 + v0.4.0 if they'd come out separately. Two big changes — a rebrand and a knowledge-base reframe — plus a lot of new MCP capability for AI assistants.

> Note for existing users: the bundle ID, MCP server name, repo names, and DMG filename all stay as `meetflow`/`Meetflow` for back-compat. Your TCC permissions, your `~/.claude.json` MCP config, and your `~/.local/share/meetflow/` data carry over without any action on your part. Only the user-visible name in the menu bar / About box / Settings / docs is now **Mindflow**.

## What's new

### Mindflow is now a knowledge base, not just a meeting tool

Three input methods feed one searchable archive:

- **🎙️ Meetings** — system audio + mic, live transcripts, MCP-streamed to your AI assistant.
- **⌨️ Dictation** — global hotkey ⌃⌥⌘Space, ~1.5s latency, auto-paste anywhere.
- **📁 Files & URLs** — drag, drop, paste a YouTube URL, watched folders. All local.

Everything you say, hear, or transcribe lands in one place — searchable by you and any AI assistant via MCP.

### Auto-named meetings

No more `meeting_2026-04-28T14-30-00.md` headlines. Every saved transcript gets an auto-generated `display_name` from its first content words ("Sprint planning, week 17" instead of a timestamp). You can rename any meeting — click the title in Past Meetings to edit, or use the new MCP tool from your AI assistant.

A one-time migration runs on first launch to backfill display names for every existing transcript. Audited at `~/.local/share/meetflow/migration-v0.3.log`.

### Past Meetings — cards layout

A redesigned Past Meetings view replaces the list:

- Cards grid grouped by Today / Yesterday / Last Week / Last Month / Older.
- **Time filter defaults to Last 7 days** (override or pick All time).
- Filter by folder, by labels, or text-search inline.
- Folder-color card backgrounds (deterministic from the folder name) or hash gradients for uncategorized meetings.
- Click a card to open the transcript. Right-click for Rename / Move / Add labels / Delete / Export.
- Toggle to list view if you want maximum density.

### Folders + labels

Meetings can now live in folders (real subdirectories under `~/.local/share/meetflow/transcripts/`, e.g. `clients/acme/`) and carry multi-value labels. New **Library** tab in Settings shows the folder tree, label aggregate, and a retention setting.

### 14 new MCP tools

Your AI assistant can now do a lot more:

- `search_knowledge_base` — unified substring search across meetings + dictation + batch with `within_last` time-window (default 3 months; pass `"all"` to search the entire archive).
- `search_dictation_history` / `list_dictation_history` — dictation surfaces are now visible to MCP.
- `clean_empty_meetings` — defensive cleanup of zero-segment recordings (dry-run by default; requires explicit confirm).
- `transcribe_file` — drop a path or URL into Mindflow's batch queue from your AI assistant.
- `rename_meeting` — set the display name from the AI side.
- `list_folders` / `create_folder` / `move_meeting` / `rename_folder` — folder CRUD.
- `add_meeting_labels` / `remove_meeting_labels` / `list_labels` — label CRUD.
- `delete_meeting` — single-target delete (dry-run by default; explicit confirm required).

The new `within_last` convention applies to all search/list tools — short relative offsets like `"7d"`, `"3m"`, `"1y"`, or the special value `"all"`. Defaults are intentionally bounded, not all-time, so AI search results stay relevant.

The MCP server name stays `meetflow` — no `~/.claude.json` changes needed for existing users.

### Settings — three groups

Tabs now split into three logical groups in the sidebar:

- **Input methods** — Meetings (placeholder for v0.4.1), Dictation, Files
- **Knowledge** — Library (NEW), Dictionary
- **System** — General, Permissions, Models, Integrations

## Under the hood

- New `TranscriptSource` interface in the MCP server with one reader per pillar (meetings / dictation / batch). Foundation for the v0.5 FTS index.
- LRU cache on parsed transcripts (keyed by path + mtime) — search across hundreds of meetings is now consistently fast.
- Atomic frontmatter writes everywhere a transcript is rewritten (Parakeet final pass, diarization regen, manual rename) — your display names + labels + folder never get stripped on regeneration.
- The recursive transcript walker lights up subdirectory storage immediately, so any folder you create via the new MCP tool appears in Past Meetings without app-side schema changes.
- The single self-signed code-signing certificate continues to anchor TCC permissions across releases — no Screen Recording / Microphone re-grant on update.

## Known follow-ups (for v0.4.1)

- **Meetings tab is a placeholder** — meeting-specific settings (auto-record, default model, language, translate) still live inside General. They'll move out in v0.4.1.
- **Folder rename in Settings doesn't cascade** to nested meetings' frontmatter `folder:` field. The MCP-side `rename_folder` tool DOES cascade — use that for now if you want consistent metadata.
