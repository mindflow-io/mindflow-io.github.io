# Meetflow v0.2.25

**Released:** 2026-04-27

## What's new

- **Permissions stay granted across updates.** Until now, every Meetflow update silently invalidated your Screen Recording and Microphone permissions because each build had a different code-signing fingerprint, which macOS treats as a brand-new app. Starting with this release Meetflow ships with a stable code-signing identity, so the permissions you grant once are kept across every future update. No more re-toggling under Privacy & Security every time a new version drops.

## Heads-up — one final time

Because v0.2.24 was signed with the old per-build identity and v0.2.25 is the first release with the new stable identity, macOS sees them as different apps for permission purposes. **You will need to re-grant Screen Recording and Microphone access one more time** when you update to this release. After that, you should not be asked again on subsequent updates.

If recording fails silently after updating, open System Settings → Privacy & Security → Screen Recording and Microphone, remove any old Meetflow entries, and re-grant the permissions.

## Under the hood

- Build pipeline now signs with a 4096-bit RSA self-signed certificate and pins each build's macOS Designated Requirement to the certificate root. This is the small mechanical change that gives macOS a stable identity to track permissions against.
- Preflight and postflight signing assertions added so future builds can't accidentally regress to per-build ad-hoc signing or enable hardened runtime, both of which silently break the microphone permission flow.
