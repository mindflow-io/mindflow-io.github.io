# Mindflow v0.4.1

**Released:** 2026-04-30

A hotfix for three v0.4.0 regressions plus a Settings polish pass. If you upgraded to v0.4.0 and your Past Meetings looked empty, this release brings them back.

## Fixes
- Past Meetings now shows your full library again. v0.4.0 had a path-handling bug that hid every transcript whose data lived under `~/.local/share/meetflow/` — the standard install location, so this affected everyone. Your meetings were never lost; they just weren't being read.
- The default time-window filter for Past Meetings is now All time. v0.4.0 defaulted to Last 7 days, which made libraries with older meetings look empty until you noticed the filter chip.
- Settings → Library → "+ New folder" now works and shows clear inline errors when a name is invalid (empty, dot-prefixed, or duplicate). The previous silent failure was a side effect of the same path-handling bug.

## What's new
- Settings → Library → Retention is now a friendly drop-down with curated presets (Forever / 30 days / 90 days / 180 days / 1 year / 2 years) plus a Custom… option that accepts any positive day-count. The "0 = Forever" semantic is now a first-class label instead of a rigid +/- counter.
- The Past Meetings empty state now distinguishes "library is empty" from "filters are hiding meetings" and offers a single-click button to clear filters and show everything.

## Under the hood
- Removed the empty Settings → Meetings placeholder tab. Meeting controls continue to live on the General tab.
- Renamed the Settings sidebar group "Input Methods" → "Voice Input" now that only Dictation + Files remain in it.
