# Mindflow v0.4.2

**Released:** 2026-05-01

Library workflow polish: bulk actions on Past Meetings, a duration filter, a unified folder list between Settings and the Move sheet, and bigger default window sizes. Two MCP fixes round out the release.

## What's new
- Past Meetings cards now support bulk select. Click the checkbox on one card and a sticky action bar appears with Move, Add labels, Delete, and Deselect actions. Operations apply to every selected card in one pass.
- New Duration filter chip on Past Meetings with presets (under 1 minute, 1–5, 5–15, 15–30, 30–60, over 1 hour) and a Custom bounds sheet where you can type exact minute ranges.
- Default window sizes are roughly doubled. The menu bar popover, Past Meetings window, Settings, and onboarding are all more comfortable to work in on modern displays.
- Settings → Library → Folders and the Move-to-folder sheet on meeting cards now show the same folder list, with case-insensitive snap-to-existing. Typing `experio` when you already have a folder named `Experio` lands the meeting in `Experio` instead of creating a sibling, and an inline hint warns you before you commit.

## Fixes
- `list_past_meetings` with `within_last: "7d"` now honors the requested window. A meeting recorded 17 days ago whose file was touched more recently (by a rename, move, or label edit) used to slip past the filter because the window compared against mtime instead of the recorded timestamp; it now compares against the recording time parsed from the filename and frontmatter.
- `search_knowledge_base` and `add_meeting_labels` no longer reject `limit` and `labels` with "expected number/array, received string" errors from some MCP clients. Numeric and array parameters now tolerate JSON-encoded strings so AI assistants can invoke those tools reliably.

## Under the hood
- Library folder enumeration is now centralized in app state, so Settings and the Move sheet share one list and can never drift. Added an explicit case-insensitive resolver used by every folder-create / folder-move code path.
- Added a small zod preprocessor wrapper for every array/number/boolean tool parameter, as a defensive layer for MCP clients that stringify arguments before transport. Existing clients that send native types are unaffected.
