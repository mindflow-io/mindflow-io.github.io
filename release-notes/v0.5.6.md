# Mindflow v0.5.6

**Released:** 2026-05-05

## What's new
- Mindflow now warns you before quitting if a meeting is still recording or finalizing, so a misclick on Quit (or a stray Cmd+Q) can no longer drop the in-flight transcript.
- The warning offers three choices: "Stop & Save, Then Quit" (the safe default — Enter), "Quit Anyway" if you really mean it, and "Cancel" to keep recording.
- If macOS shuts down or restarts while Mindflow is recording, the app makes a best-effort attempt to finalize the transcript before the system kills it.

## Fixes
- (none — feature release)

## Under the hood
- New `AppShutdownGuard` singleton bridges `AppState.captureState` into `applicationShouldTerminate(_:)` so the delegate can make the right call without owning a SwiftUI environment.
- `Stop & Save` waits up to 30 seconds for the finalize to complete before letting the app exit, then replies to AppKit's terminate request from the main actor.
- Re-entry guard prevents stacked alerts if a second termination request arrives while a finalize is already in flight.
