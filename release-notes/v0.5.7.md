# Mindflow v0.5.7

**Released:** 2026-05-05

## Fixes
- Fixed a crash that could occur shortly after stopping a recording. Quitting Mindflow during the post-meeting speaker-identification or transcript-enhancement step is now safe.

## What's new
- Mindflow now also asks for confirmation if you try to quit while a dictation is in progress, the same way it does for recordings. The warning is honest about whether a recording, a dictation, a transcript finalize, or a combination is currently running.
- "Stop & Save, Then Quit" now also stops an in-flight dictation cleanly.
- If macOS shuts down or restarts while a dictation is active, Mindflow makes a best-effort attempt to cancel it cleanly before the system kills the process.

## Under the hood
- The shutdown guard now tracks four in-flight workloads instead of one: meeting recording, dictation, enhanced diarization, and the Parakeet final pass. Quitting during any of them now blocks (or warns) instead of crashing.
- The "wait for finalize" deadline was bumped from 30 to 60 seconds because cold CoreML model loads can take 20–40 seconds on the first run after launch.
- Alert wording is now shape-aware — separate phrasings for recording-only, dictation-only, finalize-only, recording+dictation, and a generic fallback if multiple flags are set at once.
