# Mindflow v0.5.5

**Released:** 2026-05-04

## What's new
- New meeting transcripts now record four durable facts in the YAML frontmatter:
  `duration_seconds`, `segment_count`, `started_at`, `ended_at`. These are
  written once at finalize time and survive renames, label changes, and
  folder moves — so cleanup, search, and listing tools never have to recompute
  them from segment timestamps.

## Fixes
- "Clean empty meetings" no longer reports legacy transcripts as zero-duration.
  When the new frontmatter `duration_seconds` is missing, the cleanup step now
  falls back through (1) segment timestamps and (2) the `**Duration**: MM:SS`
  line in the markdown body, in that order. Old recordings with hundreds of
  segments but no parseable timestamps now report their real duration.

## Backward compat
- Pre-v0.5.5 transcripts continue to work without rewrite. Renaming or moving
  one of those files (via the Library tab or `rename_meeting` / `move_meeting`)
  will NOT auto-stamp the new fields — they only appear on freshly-recorded
  meetings going forward. A future migration can backfill if needed.
