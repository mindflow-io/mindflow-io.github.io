# Mindflow v0.5.1

**Released:** 2026-05-04

## What's new
- `transcribe_file` now accepts YouTube and other public URLs. Audio is fetched via the bundled `yt-dlp`, transcoded to 16 kHz mono WAV, and run through whisper just like a local file.
- Per-job overrides on `transcribe_file` (language, model, folder, labels) are now honored. Pass them through MCP and they land on the resulting transcript's frontmatter and on disk in the right folder.
- Documentation: full schema tables for all 22 MCP tools are now on https://mindflow.mintlify.app.

## Fixes
- Settings window no longer shows a phantom empty band at the top when the window loses focus.
- URL `transcribe_file` jobs that yt-dlp can't fetch (or that aren't supported) move to a `unsupported/` sibling folder so the watcher stops re-processing them on every sweep.

## Under the hood
- `BatchTranscriberConfig` cache is invalidated when a per-job override changes the (model, language, translate, diarize) tuple. Without this, an override would silently fall back to the warm-cache instance's settings.
- yt-dlp invocation switched to `--audio-format wav --postprocessor-args "ffmpeg:-ar 16000 -ac 1"`. Requires `ffmpeg` on your `PATH` (most users already have it via `brew install ffmpeg`; `brew install yt-dlp` pulls it in transitively).
- Old `meetflow-ai/*` GitHub repos are archived now that the rebrand has settled.
