# Mindflow v0.5.0

**Released:** 2026-05-03

The first release under the new **Mindflow** brand. This is a meaningful upgrade from v0.4.x — please read the install notes below before upgrading.

## What's new

- **Same-speaker segment merging** in markdown exports — your transcripts now read as conversation, not a wall of timestamps. Default ON; toggle in Settings → Transcription.
- **Configurable display name** for "[You]" — set your name once in Settings → Transcription, and exports use it everywhere (e.g. `[Mohamed]: ...`).
- **Bug-report flow** — a "Report a Bug" button in Settings → About, the Crash dialog, and the menu bar footer captures the last hour of diagnostic logs and opens a pre-filled GitHub issue. Logs contain only Mindflow's own diagnostics — no transcript content.
- **MCP `transcribe_file` works end-to-end** — drop a job from your AI assistant and the app picks it up via a watched batch queue. Result lands in your transcripts folder as `batch_*.md`.
- **Live transcript confidence signals** — Whisper now emits per-segment `avg_logprob` so the validity scorer can suppress low-confidence hallucinations as they arrive.
- **VoiceOver labels** added to dictation + transcription settings controls for accessibility.

## Fixes

- Live transcript files now preserve correct ordering between segments and scrubber events under load.
- `rename_meeting` MCP tool now works for files inside subfolders (was broken for any folder-organized meeting).
- `add_meeting_labels` / `remove_meeting_labels` MCP tools fixed (path validator was hardcoded to the legacy `meetflow` data dir).
- Whisper-cli probe at app launch now allows up to 15 seconds for cold-cache Metal kernel compilation (was killing the app at 2s on Apple Silicon).
- Settings window no longer shows an empty padding band at the top when unfocused.
- Auto-naming no longer picks "-" when the mic chunk is silent.
- Security: closed a path-validator bypass via symlinked `~/Downloads`.

## Under the hood — read this before installing

This release is a full rebrand from "Meetflow" to "Mindflow". You'll need to do a few things once:

1. **Move your data** (run in Terminal *before* installing the new version):
   ```bash
   mkdir -p ~/.local/share/mindflow ~/Library/Caches/Mindflow
   mv ~/.local/share/meetflow/* ~/.local/share/mindflow/ 2>/dev/null
   mv ~/Library/Caches/Meetflow/* ~/Library/Caches/Mindflow/ 2>/dev/null
   ```
2. **After installing v0.5.0**, drag the old `/Applications/Meetflow.app` to the Trash. Mindflow installs as a separate app at `/Applications/Mindflow.app`.
3. **Re-grant permissions** in System Settings → Privacy & Security: Screen Recording, Microphone, Accessibility. The bundle ID changed (`ai.meetflow.app` → `ai.mindflow.app`), so macOS asks again.
4. **MCP / Claude Code users**: open Settings → Integrations → click "Set Up" to refresh your Claude Code config. The new entry registers as `mcpServers.mindflow` with tool prefix `mcp__mindflow__*` and slash command `/mindflow`. Manually delete the old `"meetflow"` entry from `~/.claude.json`.

## Known limitations (v0.5.1 follow-ups)

- `transcribe_file` MCP doesn't apply per-job `language`/`model`/`folder`/`labels` overrides yet — uses your configured defaults.
- `transcribe_file` MCP with an `http(s)://` URL silently no-ops. Workaround: drag the URL into the menu bar popover.
