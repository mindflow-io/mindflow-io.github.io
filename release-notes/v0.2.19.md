# Meetflow v0.2.19

**Released:** 2026-04-14

## What's new

- **Critical updates.** Emergency releases can now show an undismissable red "Critical update required" banner instead of the polite blue one. When Meetflow finds a serious bug between releases (data loss, security, silent recording failure), the next update is marked critical server-side and your Meetflow will nag you until you click Update now. For normal releases — which is almost every release — nothing changes. You still get the friendly dismissable banner.

- **Richer update metadata.** The updater now also pulls in a "What's new" link (the page you're reading right now) alongside the version number.

## Under the hood

- A new `VERSION.json` sidecar alongside the plain `VERSION` file lets us ship more information to the updater without breaking older clients. Old Meetflow builds keep working; new ones read the richer metadata.
- Graceful fallback: if the sidecar is missing or malformed, the updater falls back to the plain text file — no regression risk.

Nothing user-visible was removed or changed in day-to-day use. Just click Start Recording and go.
