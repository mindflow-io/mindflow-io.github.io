# Meetflow v0.2.20

**Released:** 2026-04-15

## Critical fix — please update

If you tried to record a meeting today and watched the "0 segments captured" counter refuse to move no matter how long you talked, this release is for you. A recent Homebrew update to `whisper-cpp` silently broke transcription on every machine that received it: Meetflow would capture audio correctly, cut it into chunks, and hand each chunk to whisper — and whisper would abort before processing a single sample. The root cause was a single command-line flag we depended on getting renamed upstream, which made whisper-cli reject our invocation and exit before it saw your audio. Fixed by migrating to the new flag spelling, and this release is marked critical so every Meetflow that auto-checks will nag you to update.

## What's new

- **Arabic language support.** Settings → Recording now has Arabic in the language picker, with its own recommended model (`large-v3` — the best open-source Arabic STT that runs fully locally, currently ranked #3 on the Open Universal Arabic ASR Leaderboard). When you pick Arabic, the Models tab filters to only the models that actually support it and the "Recommended for Arabic" pill lights up on the right one. Meetflow also seeds whisper with an Arabic framing sentence to sharply bias the recognizer away from English fallback noise on conversational speech — a known footgun for Arabic dialects.
- **"Recommended for multilingual" badge.** Spanish, French, German, Japanese, and Chinese now surface `large-v3-turbo` as the recommended model with a matching badge, in both Settings and the menu bar model picker. Turbo is a 5–8× faster drop-in replacement for full large-v3 with comparable accuracy on high-resource languages.
- **Language and model are shown and locked while recording.** The red Recording card now shows the exact language and model that's feeding the current session, right next to the timer, with a lock icon. The Language picker and the "Use" model buttons are disabled for the duration of the meeting so you can't accidentally swap mid-session — whisper would otherwise produce a transcript that visibly changes quality or language halfway through.
- **Stopping a meeting shows a Finalizing card and auto-opens the transcript.** Clicking Stop Recording no longer just snaps back to the empty menu bar. Meetflow shows a "Finalizing…" spinner while queued audio chunks drain through whisper, then pops up a window with the transcript that just finished. No more "did it save?" moments.
- **5-step first-run onboarding.** New installs now get a proper welcome flow: who Meetflow is, how it works, macOS permission setup, language and model choice (with Arabic models badged if you pick Arabic), and a one-click "Set Up Claude Code" button that installs the `/meetflow` slash command and registers the MCP server for you. Re-runnable from Settings → General → "Show onboarding again".
- **Crash visibility.** Both Meetflow-itself crashes and whisper-cli child crashes now surface as an in-app dialog on next launch instead of dying silently. Meetflow scans `~/Library/Logs/DiagnosticReports` for the matching crash file, parses the exception type, signal, and backtrace, and persists a combined report for follow-up.
- **MCP `control_meeting start` gains language and model parameters.** Your AI assistant can now start a recording with `language: "es"` or `model: "large-v3"`. Pass only `language` and Meetflow auto-selects the best-recommended model for it; pass `model` to override. Useful when an assistant knows ahead of time what language the meeting will be in.

## Fixes

- **Transcription finally works again** (see the critical fix above).
- **Control-from-MCP reliability.** Starting or stopping a meeting from Claude Code had a subtle window where MCP could report success but the Meetflow app would never actually act on the command. Two changes fix this for good: MCP now writes commands atomically (tmp-file plus rename) so the Swift-side directory watcher reliably fires, and MCP now waits for the app to actually acknowledge the state change before claiming success. If anything downstream fails, you get a real error instead of a silent optimistic pass.
- **Model display consistency.** The menu bar popover and Settings → Models now show the same friendly name and description for every model, and the "Active: …" label resolves through the same code path as the language-mismatch banner — they can no longer disagree with each other about which model is live.
- **Onboarding step 5 was pointing at a dead `npm install` command** against a private registry that real users can't reach. Replaced with a one-click "Set Up Claude Code" button in the wizard itself, using the same installer Settings → Integrations uses.
- **52 MB of leaked DMG installers reclaimed.** Every Meetflow that has auto-updated since 0.2.17 was leaving old DMGs behind in `~/Library/Caches/Meetflow/`. A one-shot migration runs on first launch of this build, keeps only the current version, and deletes the rest.
- **Top-left menu bar title shows "Meetflow" again** whenever a window (Past Meetings, Transcript, Settings, Crash Report) is frontmost. The app-wide main menu had been silently missing on every window except the first-run onboarding one.

## Under the hood

- Command dispatch between MCP and the menu bar app now uses atomic-rename writes so the Swift-side parent-directory watcher reliably fires on every command. In-place truncating writes (Node's default for `writeFileSync`) don't add or remove a directory entry and go silent against `EVFILT_VNODE` on a directory fd — this was the underlying cause of the MCP sync regressions that kept recurring over the last few releases.
- Disk write pressure on live-session files dropped noticeably: the status heartbeat now reuses a persistent `FileHandle` with seek/truncate/write instead of repeatedly atomically rewriting the file, and a new `DiskWriteBudget` logs a warning if any category exceeds 300 writes/minute.
- ScreenCaptureKit audio capture now registers a no-op video output handler alongside the audio one. Without it, SCStream logs 1 Hz "VideoQueue dropping frame" errors for the whole session — harmless noise, but it drowned out every other diagnostic signal while we were chasing the whisper regression above.
