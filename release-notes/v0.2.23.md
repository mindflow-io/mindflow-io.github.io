# Meetflow v0.2.23

**Released:** 2026-04-17

## Fixes

- Recording could silently fail to capture any audio if two start commands arrived in quick succession — for example when Claude Code retried `control_meeting start`, or when a command file was double-written during onboarding. The app now serializes its startup work across concurrent requests, so rapid starts produce a single clean recording instead of a frozen UI and an empty transcript.
- The Transcripts window sometimes got stuck on "Loading transcript…" with no way to recover other than force-quitting the app. This was the same underlying issue — fixed by the same change.
- The menu-bar popover occasionally showed "Start Recording" while the app was actually active. The popover now reflects the true recording state immediately.

## Under the hood

- Speech recognizer preflight runs off the main thread with a 30-second timeout and graceful SIGTERM → SIGKILL escalation, so a stuck GPU init can no longer freeze the app.
- `AppState.startRecording` is now a single-flight task — concurrent callers observe the in-flight task's terminal state instead of spawning duplicate pipelines.
- Landing page revamped: clearer Fathom comparison, self-hosted fonts for better privacy, a developer section, and an FAQ.
