# Meetflow v0.2.18

**Released:** 2026-04-13

## Highlights — today's flagship bug fix

This release closes the "MCP/app sync" bug class that had been regressing through three separate earlier fixes. Root cause was a narrow window during recording startup where `status.json` could be deleted and not immediately rewritten, which made MCP's "is the app running?" check fail and fall through to a legacy code path that never signalled the app to stop.

- **Status heartbeat no longer skips during active recording.** Writes `state:"active"` every 5 seconds while recording, `state:"idle"` while not. Guarantees MCP's freshness check always sees the app as alive.
- **`LiveSessionWriter.startSession` no longer deletes status.json.** Atomically overwrites it with `state:"active"` so there's no window where MCP sees "no file" and decides the app is absent.
- **MCP `handleStop` defensive fallback.** Added a `hasRecentSessionActivity()` check (session.jsonl mtime within 30s) so even if status.json is stale/missing, MCP can still route a stop command through `FileCommandWriter.sendCommand`.
- **42% idle CPU burn eliminated.** Three teardown leaks fixed in `AudioCaptureKit`:
  1. `AudioTapManager.stop()` is now async, calls `removeStreamOutput` before `stopCapture`, and sets `self.stream = nil` last. `ScreenCaptureKit` no longer keeps delivering audio buffers to a dead handler.
  2. `MeetingCaptureEngine.stop()` actually awaits the tap manager teardown in all three callers (normal stop, start-error path, signal handler) instead of fire-and-forget.
  3. `micEngine.inputNode.removeTap(onBus: 0)` added — `AVAudioEngine.stop()` does not do this implicitly.
  4. `silenceCheckTask` is stored and explicitly cancelled in `stopRecording()` instead of relying on a polling flag check.
- **Past Meetings opens instantly.** Transcript detail view now async-loads the file off the main thread and renders segments in a `LazyVStack` instead of one giant `Text`. Large meetings that used to take ~10 seconds now open in under 200 ms.
- **Command.json watcher survives inode replacement.** Watches the parent directory instead of the single file, dedupes by timestamp, so atomic-rename file rotation no longer invalidates the file-descriptor-backed watcher.
- **Bundled libwhisper.1.dylib ships correctly.** Build script was using `find -type f` which silently skipped symlinks (Homebrew ships libwhisper as a symlink). Users without Homebrew no longer hit a dyld crash on first launch.
- **New multilingual + Arabic speech models** in the Settings → Models catalog.

## Breaking and schema changes

- `StatusOutput` now includes a `timestamp` field (additive, backwards-compatible).
